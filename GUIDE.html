How We Built PostFlow

A complete guide for anyone who wants to understand this project from the very beginning — written in plain language, no experience needed.

What is this app?

PostFlow is a website that helps you write marketing posts for LinkedIn and WordPress.

You type a topic, click a button, and an AI writes the post for you. Then you can edit it, approve it, copy it, and paste it into LinkedIn or WordPress yourself.

Simple idea — but we needed to build a lot of pieces to make it work.

What you need before starting

1. Node.js

Node.js lets your computer run JavaScript outside of a browser. Think of it like an engine that powers your project.

  • Download from nodejs.org
  • Pick the version that says LTS — that means "stable and reliable"
  • Install it like a normal program

To check it worked, open Command Prompt (search for cmd in the Start menu) and type:

node --version

If you see a number like v22.0.0, you're good.

⚠ Use Command Prompt, not PowerShell
PowerShell blocks npm scripts on Windows. Always use cmd.

2. A code editor

This is where you write and read code. We recommend Visual Studio Code — it's free.

  • Download from code.visualstudio.com

3. A Google account

You need one to set up Firebase (the database). You probably already have one.

Part 1

The starting point

We started with just one file: marketing-post-manager.jsx.

This file already had the full app written in it — buttons, screens, logic for talking to AI and Firebase. But it had two problems:

Problem 1 — Firebase loaded from a URL
import { initializeApp } from "https://www.gstatic.com/firebasejs/10.12.0/firebase-app.js";

This works in a plain browser, but not in a proper development setup with build tools.

Problem 2 — Secret keys written in the file
const CONFIG = {
  CLAUDE_API_KEY: "YOUR_CLAUDE_API_KEY",
  ...
}

If you share the file, you share your keys. Anyone can use them and charge costs to your account.

Our job was to fix both problems and turn this single file into a proper project.

Part 2

Setting up the project tools

What is Vite?

Vite is a tool that:

  • Lets you run your app locally at http://localhost:5173
  • Automatically refreshes the browser when you save a file
  • Builds your app into a finished website when you're ready to publish

What is React?

React is the library that makes the app's buttons and screens work. The .jsx file extension means the file uses React.

What is Tailwind CSS?

Tailwind is a styling tool. Instead of writing a separate CSS file, you add short words like text-blue-600 or rounded-xl directly onto elements, and it figures out the styling.

Setup steps

Step 1 — Create package.json

This file is the shopping list. It tells npm which tools to download.

{
  "name": "marketing-post-manager",
  "private": true,
  "version": "1.0.0",
  "type": "module",
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview"
  },
  "dependencies": {
    "firebase": "^10.12.0",
    "react": "^18.3.1",
    "react-dom": "^18.3.1"
  },
  "devDependencies": {
    "@tailwindcss/typography": "^0.5.15",
    "@vitejs/plugin-react": "^4.3.1",
    "autoprefixer": "^10.4.20",
    "postcss": "^8.4.47",
    "tailwindcss": "^3.4.14",
    "vite": "^5.4.10"
  }
}
  • dependencies — tools needed when the app is running
  • devDependencies — tools only needed while building
  • scripts — shortcuts. npm run dev runs the "dev": "vite" line.

Step 2 — Create vite.config.js

Tells Vite to use the React plugin so it understands .jsx files.

import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";

export default defineConfig({ plugins: [react()] });

Step 3 — Create tailwind.config.js

Tells Tailwind which files to scan for styling words. The @tailwindcss/typography plugin makes HTML blog posts look nice.

export default {
  content: ["./index.html", "./*.jsx"],
  theme: { extend: {} },
  plugins: [require("@tailwindcss/typography")],
};

Step 4 — Create postcss.config.js

PostCSS is a helper that processes CSS. Tailwind needs it to work.

export default { plugins: { tailwindcss: {}, autoprefixer: {} } };

Step 5 — Create index.html

The blank webpage. The <div id="root"> is an empty box — React fills this box with all the buttons and screens.

<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>PostFlow — Marketing Post Manager</title>
  </head>
  <body>
    <div id="root"></div>
    <script type="module" src="/main.jsx"></script>
  </body>
</html>

Step 6 — Create index.css

Three lines that activate Tailwind CSS.

@tailwind base;
@tailwind components;
@tailwind utilities;

Step 7 — Create main.jsx

The power button. It finds the empty #root box and puts the app inside it.

import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import "./index.css";
import App from "./marketing-post-manager.jsx";

createRoot(document.getElementById("root")).render(
  <StrictMode>
    <App />
  </StrictMode>
);

Step 8 — Install everything

Open Command Prompt in the project folder and run:

npm install

This reads package.json and downloads all tools into a node_modules folder. Takes about a minute.

Part 3

Moving secret keys to .env

Why does this matter?

If your API key is in the code file and you share it, whoever receives it can use it to generate AI content — and charge it to your account.

A .env file keeps secrets separate. The .gitignore file makes sure Git never uploads it. So your keys stay safe.

How it works

Vite reads the .env file and makes values available in your code using import.meta.env.VITE_...

Before (dangerous)After (safe)
CLAUDE_API_KEY: "sk-ant-api03-..." CLAUDE_API_KEY: import.meta.env.VITE_CLAUDE_API_KEY

The .env file (which is never shared) contains:

VITE_CLAUDE_API_KEY=sk-ant-api03-...
Important rule
Every variable you want to use in your app must start with VITE_. Variables without that prefix stay hidden — Vite won't expose them to the browser.

The .env.example file

This is a blank copy of .env with no real values. You commit this to Git so other people know which variables they need to fill in.

Part 5

What is Firebase?

Firebase is a service made by Google. It gives you a database without having to build one yourself. We use two parts:

🗄️
Firestore
Stores your posts. Think of it like a shared Google Sheet that your app can read and write to.
🖼️
Firebase Storage
Stores image files. When you upload an image, it goes here and the app gets back a URL to use in the post.

How to set up Firebase

  1. 1Go to firebase.google.com and sign in with your Google account.
  2. 2Click Add project and give it a name.
  3. 3Click the Web icon (</>), give it a nickname, click Register app.
  4. 4Copy each value from the firebaseConfig block into your .env file.
  5. 5In the left menu: Build → Firestore Database → Create database → Test mode → Enable.
  6. 6In the left menu: Build → Storage → Get started → Test mode → Enable.
⚠ Test mode expires after 30 days
Go to Firebase Console → Firestore → Rules and extend the expiry date, or set proper security rules.
Part 6

Fixing the AI API — the CORS problem

What is CORS?

CORS stands for Cross-Origin Resource Sharing. It's a security rule built into every browser.

When your app (at localhost:5173) tries to talk to the Anthropic API (at api.anthropic.com), the browser says: "You're from a different address — I won't allow this."

The solution: a proxy

A proxy is a middleman. Instead of the browser calling Anthropic directly:

Browser
→
Anthropic API ✗

We tell the browser to call Vite's own server, which forwards the request to Anthropic:

Browser
→
Vite proxy
→
Anthropic API ✓

Because Vite's proxy is server-side (not in the browser), CORS doesn't apply. We also remove the origin and referer headers so the API doesn't detect the browser origin.

server: {
  proxy: {
    "/api/anthropic": {
      target: "https://api.anthropic.com",
      changeOrigin: true,
      rewrite: (path) => path.replace(/^\/api\/anthropic/, ""),
      configure: (proxy) => {
        proxy.on("proxyReq", (proxyReq) => {
          proxyReq.removeHeader("origin");
          proxyReq.removeHeader("referer");
        });
      },
    },
    "/api/openai": { /* same pattern */ }
  }
}

In the app code, the URL changes from https://api.anthropic.com/v1/messages to /api/anthropic/v1/messages.

Part 7

The AI generation system

Two providers

ProviderCompanyModel
ClaudeAnthropicclaude-sonnet-4-6
GPT-4oOpenAIgpt-4o

Both work the same way: you send them a message, they send back text.

What is a system prompt?

A system prompt is instructions you give the AI before the user's message. It sets the personality and rules.

  • LinkedIn: "You are an expert LinkedIn content creator. Write engaging professional posts…"
  • WordPress: "You are an expert blog writer. Write a full blog post in HTML…"

What is tone?

Tone changes how the writing sounds. The tone instruction is added to the system prompt before sending.

ToneWhat it means
ProfessionalFormal and authoritative
CasualFriendly and relaxed, like talking to a friend
InspirationalMotivating and uplifting
BoldShort, punchy, direct — strong statements
Part 8

The "publish" decision

Originally the app had real LinkedIn and WordPress publishing — it would automatically post for you.

We turned this off and replaced it with clipboard copy instead.

Why clipboard?
Setting up LinkedIn and WordPress API connections requires many steps and credentials. For now, it's simpler to click "📋 Copy", open LinkedIn, and paste manually. The full API publishing can be re-added later.

Part 11

The start.bat shortcut

Normal users shouldn't have to open Command Prompt every time. So we made a .bat file — double-clicking it starts everything automatically.

@echo off
cd /d "%~dp0"
echo Starting PostFlow...
start "" cmd /k "npm run dev"
timeout /t 3 /nobreak >nul
start "" "http://localhost:5173"
LineWhat it does
@echo offStops the script printing every command it runs
cd /d "%~dp0"Goes to the folder where the .bat file lives
echo Starting PostFlow...Shows a message so you know it's working
start "" cmd /k "npm run dev"Opens a new window and starts the server
timeout /t 3 /nobreak >nulWaits 3 seconds for the server to start
start "" "http://localhost:5173"Opens the app in the browser

Running the app

First time only

npm install

Every time after

Double-click start.bat — or in Command Prompt:

npm run dev

Then open: http://localhost:5173

To stop

Close the Command Prompt window that has npm run dev running in it.
